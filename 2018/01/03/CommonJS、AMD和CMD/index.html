CommonJS、AMD和CMD

Web JavaScript
 2018/01/03

是什么

CommonJS是作用在服务器端，同步的，如nodeJS。根据CommonJS的规范，一个文件就是一个模块，加载的时候用require方法，然后执行之后返回文件内部的exports对象

AMD是Asynchronous Module Definition的缩写，意思是“异步模块定义”

CMD是Commom Module Definition的缩写，意思是“通用模块定义”

AMD是RequireJS对模块定义的规范化产物

CMD是SeaJS对模块定义的规范化产物

区别

AMD和CMD最大的区别就是对依赖的执行时机处理不同（不是加载的时机或者方式）

  • AMD推崇依赖前置，在模块定义的时候就要声明依赖的模块
  • CMD推崇就近依赖，在用到某个模块的时候再去require
  • AMD的API默认是一个当多个用，CMD的API职责单一比较纯粹

简要概述

AMD和CMD的区别是对依赖的执行时机处理不同，不是加载的时机。不是字面意思AMD异步，CMD同步，它们都是异步加载。

AMD在加载模块完成以后就会执行该模块，所有的模块加载完成以后都会进入require的回调函数执行主逻辑函数。这样最终依赖模块的执行顺序和书写顺序不一致，哪个先被网络请求下来就先执行哪一个，但是最终的主逻辑一定会是在所有的依赖加载完成之后才会被执行。

CMD加载完某个依赖模块以后并不立即执行，只是下载下来而已，在完成所有的依赖模块加载之后进入主逻辑，遇到require语句的时候才会执行相应的模块，这样模块的执行顺序和模块的书写顺序就完全一致了。
